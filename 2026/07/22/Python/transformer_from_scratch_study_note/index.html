这次手写 Transformer 时，最容易让人卡住的不是某一个类，而是同一份数据在不同模块中的形状、来源和可见范围。把这三件事统一后，Embedding、多头注意力、Encoder 和 Decoder 就能连成一条可检查的数据流。

从张量形状开始

Notebook 使用 batch_first=True 的约定，序列表示统一写成：

  • batch 是一次并行处理的样本数；
  • sequence 是 token 数或时间步数；
  • d_model 是每个 token 的特征维度。

例如 (128, 64, 512) 表示一个 batch 有 128 条序列，每条有 64 个 token，每个 token 已经表示成 512 维向量。先固定这个约定，后面看到注意力分数 (batch, heads, query_len, key_len) 时，才知道最后两个维度分别在比较谁和谁。

Embedding 与位置编码

输入给模型的是整数 token id，nn.Embedding 负责把它查表映射为连续向量：

这样 token 张量从 (batch, sequence) 变成 (batch, sequence, d_model)。注意力本身不能区分 token 的原始排列顺序，所以还要把位置编码加到 embedding 上。经典 Transformer 的正弦位置编码是：

$$ PE_{pos,2i}=\sin\left(pos/10000^{2i/d_{model}}\right) $$

$$ PE_{pos,2i+1}=\cos\left(pos/10000^{2i/d_{model}}\right) $$

实现时先预计算形状为 (max_len, d_model) 的位置表，前向时切出当前 sequence 长度，再与 token embedding 相加。每个位置的表示因此同时带有词义和位置信息。

多头注意力

给定输入表示 X，三个线性层得到 Q、K、V：

$$ Q=XW_Q,\qquad K=XW_K,\qquad V=XW_V $$

Query 表示当前位置要查询的信息，Key 表示各位置可匹配的特征，Value 是匹配后需要汇聚的内容。缩放点积注意力为：

$$ \operatorname{Attention}(Q,K,V)= \operatorname{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

除以 $\sqrt{d_k}$ 是为了限制点积随维度增大而扩大，避免 Softmax 太早饱和。多头注意力不是重复计算同一件事，而是将 (batch, sequence, d_model) 拆成多个表示子空间：

Mask：先统一语义

本次实现中，布尔 mask 的语义统一为：True 表示可见，False 表示屏蔽。先固定这一点，masked_fill(~mask, ...)、padding mask 和 causal mask 才不会写反。

被屏蔽的分数需要填入当前 dtype 的极小值，而不是正数。还要保证每个 Query 至少有一个可见 Key；如果整行都被屏蔽，Softmax 仍可能出现 NaN。

Encoder 和 Decoder

Encoder 的输入路径是：source token id -> source embedding 与位置编码 -> 多层 self-attention 和前馈网络 -> 每个 source 位置的上下文表示。它只处理源序列内部关系。

图：保留自作者原始学习 Notebook；其原始外部来源尚未核实。

一个 Encoder layer 依次经过 self-attention、残差归一化、逐位置前馈网络和第二次残差归一化。前馈网络虽然在每个位置独立计算，但输入输出都保持 d_model，因此可以和残差分支相加。

Decoder 的输入路径是：target token id -> target embedding 与位置编码 -> masked self-attention -> cross-attention -> 前馈网络 -> 目标词表 logits。masked self-attention 使用 target_mask，因此不能偷看未来 token。

cross-attention 是 Encoder-Decoder Transformer 的连接处：Query 来自当前 Decoder 状态，Key 和 Value 来自 encoder_output，并使用 cross_mask 屏蔽 source padding。它使每个目标位置可以从源序列中读取相关信息，最终输出层再把 (batch, target_len, d_model) 映射成 (batch, target_len, tgt_vocab_size)。
